# Чек-лист готовности к защите ВКР

## 1. Программная часть

- [ ] Приложение запускается командой `python web_app.py`.
- [ ] Открывается интерфейс по адресу `http://127.0.0.1:8000`.
- [ ] Корректно строятся все примеры из `examples/`.
- [ ] В 3D-окне модель вращается и масштабируется.
- [ ] Доступен экспорт `STL`.
- [ ] Доступен экспорт отчета `CSV`.
- [ ] Сценарий проекта сохраняется и повторно загружается.

## 2. Тестовые сценарии

- [ ] Тест №1: `simple_box` (объем и масса соответствуют расчету).
- [ ] Тест №2: `bracket` (сложная составная модель).
- [ ] Тест №3: `cylinder_mount` (цилиндр + основание).
- [ ] Проверен сценарий с ошибкой (валидный вывод сообщения об ошибке).

## 3. Материалы ВКР

- [ ] Заполнены титульные поля в `docs/DIPLOMNAYA_RABOTA.md`.
- [ ] Проверены номера рисунков, таблиц и список источников.
- [ ] Сформирована финальная версия пояснительной записки (PDF/Docx по требованиям вуза).
- [ ] Подготовлены скриншоты интерфейса и результатов расчетов.

## 4. Подготовка к выступлению

- [ ] Подготовлены слайды по `docs/PRESENTATION_PLAN.md`.
- [ ] Отрепетирован доклад 7-10 минут.
- [ ] Подготовлен демонстрационный запуск по `docs/DEFENSE_DEMO_SCRIPT.md`.
- [ ] Сделаны резервные копии проекта (локально и в облаке).

## 5. Перед выходом на защиту

- [ ] Проверен запуск на том же ноутбуке и сети, где будет защита.
- [ ] Отключены обновления и лишние фоновые программы.
- [ ] Подготовлен офлайн-бэкап зависимостей (`.venv` или wheel-архив).
- [ ] Подготовлен архив проекта для передачи комиссии.
